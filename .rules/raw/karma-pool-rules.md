# Karma Pool Usage Rules
Source: SR3 p.246–248

## Buying Additional Dice
- Spend 1 Karma Pool point to add one die to a test.
- Max additional dice = base skill or attribute value (major dice pools do not count toward this limit).
- Incremental cost: 1 point for 1st die, 2 more for 2nd (total 3), 3 more for 3rd (total 6), etc. Formula: n*(n+1)/2 for n dice.

## Re-rolling Failures
- Spend 1 point to re-roll all dice that failed a test.
- Each subsequent re-roll on same test costs +1 more point (2 for 2nd re-roll, 3 for 3rd, etc.).

## Avoiding an "Oops"
- If all dice show 1s (disastrous failure), spend 1 point to turn it into a simple failure.

## Buying Successes (Burn)
- Permanently burn 1 Karma Pool point to gain one automatic success.
- Requires at least one natural success on the test.

## Hooper-Nelson Rule (Burn, GM optional)
- Burn points to lower a TN of 12 or higher.
- Cost: 1 point per 1 point of TN reduction, minimum TN 8.

## Hand of God (Burn)
- Burn entire Good Karma + Karma Pool stock (minimum 1) to avoid certain death.
- One-time-only use.

## Refresh Rules
- Karma Pool does NOT refresh every Combat Turn like major dice pools.
- Refresh determined by GM based on scenes/time — typically every 24 hours of game time.
- Standard play: refreshes at beginning of each gaming session.
- Extended actions (enchanting, deck programming): does not refresh until action completes.
- Burned points are permanently removed and do not refresh.

## Storyteller Screen
- Needs a way to restore regenerating (non-burned) Karma Pool points.
