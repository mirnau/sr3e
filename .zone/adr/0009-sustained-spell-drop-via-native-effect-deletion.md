# Sustained-spell drop rides native ActiveEffect deletion instead of dedicated UI

Sustained spells attach as real Foundry `ActiveEffect`s tagged with `flags.sr3e.sustainedSpellId`, so players can already end a sustained spell through Foundry's built-in Effects tab — no bespoke "drop spell" button is needed. The risk was that native deletion bypasses `dropSustainedSpell()`, leaving the `sustainedSpells` actor flag entry (and its +2 TN/Drain penalty) orphaned forever. Instead of building parallel drop UI, we registered a `deleteActiveEffect` hook (`registerSustainedSpellCleanupHook` in `module/services/spells/sustainedSpells.ts`) that strips the matching flag entry whenever a tagged effect is deleted through *any* path — native UI, macro, or `dropSustainedSpell()` itself. This keeps the flag list and the live effects in sync without a second UI surface to maintain.
