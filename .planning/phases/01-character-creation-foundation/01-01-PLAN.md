  <files>module/services/character-creation/CharacterInitializer.ts</files>
  <action>
Create CharacterInitializer class that orchestrates character initialization with selected priorities.

Methods:
- `initializeCharacter(actor, selections: CharacterCreationSelections): Promise<void>` - Performs all character initialization steps

CharacterCreationSelections type: `{ metatypeId: string; magicId: string; attributePriority: string; skillPriority: string; resourcePriority: string; age: number; height: number; weight: number; }`

Initialization steps (from CharacterCreationDialogApp.svelte handleSubmit):
1. Calculate attribute points from priority (30/27/24/21/18 for A/B/C/D/E)
2. Calculate skill points from priority (50/40/34/30/27 for A/B/C/D/E)
3. Update actor with:
   - profile: age, height, weight
   - creation points: attributePoints (minus 6 initial), activePoints (skill points)
   - attributes: all set to 1, essence to 6
   - karma pool: 1
4. Create embedded metatype item (from game.items)
5. If magic priority is A or B:
   - Create embedded magic item
   - Set flag 'sr3e.hasAwakened' to true
   - Set attributes.magic.value to 6

Use actor.update() with flattened paths like "system.profile.age"
Use actor.createEmbeddedDocuments("Item", [item.toObject()])
Use actor.setFlag("sr3e", "hasAwakened", true)

Type all parameters strictly - no `any` for actor (use proper Foundry Actor type or create interface).
Let operations fail if they error - no try/catch wrapping.
