<task type="auto">
  <name>Task 1: Add chevron controls to Attributes.svelte</name>
  <files>module/ui/actors/actor-components/Attributes.svelte</files>
  <action>
Enhance Attributes.svelte to include up/down chevron buttons for each attribute during creation mode.

**Key requirements:**
- Chevrons ONLY visible when `isCharacterCreation = true` AND `attributeAssignmentLocked = false`
- During normal gameplay (isCharacterCreation = false), chevrons are completely hidden and non-interactable
- Up chevron (increase): calls AttributeSpendingService.increaseAttribute()
- Down chevron (decrease): calls AttributeSpendingService.decreaseAttribute()
- Up chevron disabled when: canIncreaseAttribute() returns false (no points or at racial max)
- Down chevron disabled when: canDecreaseAttribute() returns false (at minimum value 1)

**Implementation approach:**
1. Import StoreManager, AttributeSpendingService, FLAGS
2. Get flag stores for isCharacterCreation and attributeAssignmentLocked
3. Get actorStore for reactivity trigger on attribute changes
4. Create reactive canIncrease/canDecrease checks using $derived
5. Add chevron buttons inside each attribute card, conditional on creation mode
6. Use Font Awesome chevron icons: fa-chevron-up, fa-chevron-down

**Pattern to follow (from existing injections):**
```typescript
const storeManager: IStoreManager = StoreManager.Instance as IStoreManager;
const isCharacterCreation = storeManager.GetFlagStore(actor, FLAGS.ACTOR.IS_CHARACTER_CREATION, false);
const attributeLocked = storeManager.GetFlagStore(actor, FLAGS.ACTOR.ATTRIBUTE_ASSIGNMENT_LOCKED, false);
const actorStore = storeManager.GetROStore(actor);
```

**Reactivity pattern:**
```typescript
const canIncreaseAttr = $derived((key: string) => {
  $actorStore; // Trigger reactivity
  return spendingService.canIncreaseAttribute(actor, key);
});
```
