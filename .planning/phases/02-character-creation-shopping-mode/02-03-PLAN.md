**Files**:
- Modify `module/ui/actors/injections/AttributePointsState.svelte`

**Implementation**:
```svelte
<script lang="ts">
   import { CreationPointsService } from "@services/character-creation/CreationPointsService";
   import { StoreManager } from "@helpers/StoreManager.svelte";
   import { FLAGS } from "@constants/flags";

   const { actor, config } = $props<{ actor: Actor; config: any }>();
   const storeManager = StoreManager.Subscribe(actor);
   const actorStore = storeManager.GetActorStore();
   const attributeLocked = storeManager.GetFlagStore(FLAGS.ACTOR.ATTRIBUTE_ASSIGNMENT_LOCKED);

   const remainingPoints = $derived(() => {
      $actorStore;
      return CreationPointsService.Instance().getRemainingAttributePoints(actor);
   });

   async function lockAttributes() {
      const confirmed = await Dialog.confirm({
         title: "Complete Attribute Assignment?",
         content: "<p>You will not be able to change attributes after this.</p><p>Continue?</p>",
      });

      if (confirmed) {
         await attributeLocked.update(() => true);
      }
   }
</script>

<div class="creation-points attribute-points">
   <h3>Attribute Points</h3>
   <div class="points-remaining">
      <span class="value">{remainingPoints()}</span>
      <span class="label">remaining</span>
   </div>
   <button type="button" onclick={lockAttributes}>
      Complete Attributes
   </button>
</div>
```

### Task 5: Implement Creation Completion Logic in ShoppingCart

**Action**: Enhance ShoppingCart component to detect when all points are spent and skip warning dialog.

**Files**:
- Modify `module/ui/actors/injections/ShoppingCart.svelte`

**Implementation**:
```svelte
async function terminateCreation() {
   const service = CreationPointsService.Instance();
   const hasUnspentPoints = service.hasUnspentPoints(actor);

   if (hasUnspentPoints) {
      const confirmed = await showWarningDialog();
      if (!confirmed) return;
   }

   // Clear creation flags
   await isCharacterCreation.update(() => false);
   await attributeLocked.update(() => false);

   ui.notifications.info("Character creation complete!");
}

async function showWarningDialog(): Promise<boolean> {
   const service = CreationPointsService.Instance();
   const attrPoints = service.getRemainingAttributePoints(actor);
   const activePoints = service.getRemainingSkillPoints(actor, "active");
   const knowledgePoints = service.getRemainingSkillPoints(actor, "knowledge");
   const languagePoints = service.getRemainingSkillPoints(actor, "language");

   return Dialog.confirm({
      title: "Finish Character Creation?",
      content: `
         <p>You have unspent creation points:</p>
         <ul>
            ${attrPoints > 0 ? `<li>Attribute points: ${attrPoints}</li>` : ""}
            ${activePoints > 0 ? `<li>Active skill points: ${activePoints}</li>` : ""}
            ${knowledgePoints > 0 ? `<li>Knowledge skill points: ${knowledgePoints}</li>` : ""}
            ${languagePoints > 0 ? `<li>Language skill points: ${languagePoints}</li>` : ""}
         </ul>
         <p>These will be discarded. Are you sure?</p>
      `,
      yes: () => true,
      no: () => false,
   });
}
```

**Verification**:
- Clicking shopping cart with 0 unspent points → no warning, immediate completion
- Clicking with unspent points → detailed warning showing which pools have points
- Confirming clears all creation flags
- Notification appears on completion

## Notes

**Integration Strategy**:
- Minimal changes to existing components (Attributes.svelte)
- Services handle all business logic
- UI components remain presentational
- Flag-based state drives all visibility and behavior

**User Experience Flow**:
1. **Dialog phase**: Select priorities (Phase 1 - already complete)
2. **Attribute phase**: Spend attribute points, lock when satisfied
3. **Skill phase**: Spend skill points across three categories
4. **Completion**: Click shopping cart to finish (with warning if points unspent)
5. **Post-creation**: Character ready to play, karma shopping available

**Flag States**:
- `isCharacterCreation = true, attributeAssignmentLocked = false` → Attribute spending phase
- `isCharacterCreation = true, attributeAssignmentLocked = true` → Skill spending phase
- `isCharacterCreation = false` → Creation complete, normal play mode

**Separation from Karma Shopping**:
- Creation shopping uses point pools from priorities
- Karma shopping (Phase 3+) uses earned karma
- Same UI (shopping cart icon) but completely different backend
- No overlap: cannot spend karma during creation, cannot spend creation points post-creation

**Phase 2 Complete**:
After this plan, character creation is fully functional from dialog → point spending → completion. The character-first foundation is solid before moving to karma advancement (Phase 3).
