# Plan 02-04: Health Component Styling & ECG Animation

## Objective

Clean up the Health component's graphical layout and bring in the cardiogram animation service from the old project. The ECG animation responds to damage levels, creating an immersive health visualization.

## Prerequisites

- Phase 2 context established
- StatCard component already applied to Penalty/Overflow (done in previous session)
- Old project reference available in main branch

## Reference Files (from main branch)

- `module/services/CardiogramAnimationService.js` - ECG wave animation
- `module/services/ElectroCardiogramService.js` - Canvas management and health integration
- `module/svelte/apps/components/Health.svelte` - Old component structure

## Tasks

### Task 1: Migrate CardiogramAnimationService to TypeScript

**File:** `module/services/health/CardiogramAnimationService.ts`

Convert the ECG animation service:
- Canvas-based ECG waveform drawing
- P-Q-R-S-T wave simulation

- Frequency/amplitude controls for damage response
- Flatline mode for death state
- Scroll animation effect

Include `lerpColor` utility function (inline or separate utility file).

**Deliverables:**
- `CardiogramAnimationService.ts` with full TypeScript typing

### Task 2: Migrate ElectroCardiogramService to TypeScript

**File:** `module/services/health/ElectroCardiogramService.ts`

Convert the ECG orchestration service:
- Canvas resize handling with ResizeObserver
- Device pixel ratio scaling
- Damage-to-pace mapping (frequency/amplitude based on injury severity)
- Integration with CardiogramAnimationService
- Penalty calculation based on stun/physical damage

**Deliverables:**
- `ElectroCardiogramService.ts` with proper typing
- Constructor accepts canvas elements directly (Svelte bind:this pattern)

### Task 3: Update Health.svelte with ECG Integration

**File:** `module/ui/actors/actor-components/Health.svelte`

Update the component to:
- Add second canvas for ECG point overlay (`ecg-point-canvas`)

- Handle flatline on death state

### Task 4: Clean Up Health Component Styling

**File:** `styles/actor/components/health.scss`

Refine the SCSS to match old project's clean layout:

## Architecture Notes

**Service Pattern:**
```
Health.svelte
  └── ElectroCardiogramService (orchestration)
        └── CardiogramAnimationService (animation)
```

**Canvas Setup:**
- `#ecg-canvas` - Bottom layer, draws the scrolling ECG line
- `#ecg-point-canvas` - Top layer, draws the moving point/needle

**Damage-to-Pace Mapping:**
| Damage Level | Frequency | Amplitude |
|--------------|-----------|-----------|
| 0 (healthy)  | 1.5       | 20        |
| 1-2          | 2         | 20        |
| 3-5          | 4         | 25        |
| 6-8          | 8         | 35        |
| 9            | 10        | 40        |
| 10+ (dying)  | 1         | 8         |
