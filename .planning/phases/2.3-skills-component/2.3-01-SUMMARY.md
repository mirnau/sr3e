# Phase 2.3 Plan 1: Skills Component Summary

**Built the Skills card for the character sheet — three-tab UI (Active/Knowledge/Language) with item reactivity, attribute grouping for active skills, and creation-mode shopping buttons wired to SkillSpendingService.**

## Performance

- **Duration:** Unknown (completed prior to session — undocumented)
- **Completed:** ~2026-03-08 (retroactively documented)
- **Files created:** 5 Svelte components + 1 SCSS file

## Accomplishments

- `Skills.svelte`: tab container with item collection reactivity (DicePools pattern), three-tab navigation (Active / Knowledge / Language), derives three filtered+sorted skill arrays from `actor.items` on `itemsUpdateTick`
- `SkillsActive.svelte`: groups active skills by `linkedAttribute` into a sorted `Map`, renders each group with a capitalized attribute header and flex-wrap card row
- `SkillsKnowledge.svelte`: flat list of knowledge skills
- `SkillsLanguage.svelte`: flat list of language skills with R/W sub-value display
- `SkillCard.svelte`: unified skill card for all three categories — name, rating, specializations (read-only display), shopping `+`/`−` buttons in creation mode wired to `SkillSpendingService`; skill editor launched via icon
- `styles/actor/components/skills.scss`: clipped-corners card system matching AttributeCard visual style; tab chrome; attribute group headers; neon border specialization cards; add-spec button; shopping cart pulse animation
- `CharacterSheetApp.svelte`: Skills card registered in Packery grid

## Decisions Made

- `inCreationMode` not passed as flag store prop — SkillCard reads creation state via service rather than component prop chain
- `groupedSkills` implemented as `$derived(() => ...)` returning a function (Svelte 5 pattern for derived that returns a value)
- Item reactivity via `itemsUpdateTick` bump — same pattern as DicePools, avoids stale item arrays

## Issues Encountered (in subsequent polish session)

- `$specializationsStore` and `$readWriteStore` possibly null inside `#if` guards — fixed with `?? 0`, `?.length ?? 0`, `?? []` operators
- `onDestroy` import left unused after Svelte 5 rune migration — removed
- SpecializationCard buttons overlapping card text — fixed with `specialization-card-row` flex wrapper
- `skill-add-spec-btn` using nonexistent `stat-card` wrapper — replaced with standalone neon-border button
