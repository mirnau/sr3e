<task type="auto">
  <name>Task 1: Skills.svelte — tab container with item reactivity</name>
  <files>module/ui/actors/actor-components/Skills.svelte</files>
  <action>
    Create presentational Skills.svelte with:

    Props: `{ actor }` (same as DicePools.svelte pattern).

    Item reactivity (copy pattern from DicePools.svelte exactly):
    - `itemsUpdateTick = $state(0)` with bump function
    - `collection.on("update"|"create"|"delete", bump)` + cleanup
    - `Hooks.on("updateItem"|"createItem"|"deleteItem", bumpHook)` + cleanup
    - bumpHook checks `doc?.parent?.id === actor?.id`

    State:
    - `activeTab = $state<"active" | "knowledge" | "language">("active")`
    - `isCharacterCreation = $state<Writable<boolean> | null>(null)` — storeManager.GetFlagStore(actor, "isCharacterCreation", false)
    - `attributeAssignmentLocked = $state<Writable<boolean> | null>(null)` — storeManager.GetFlagStore(actor, "attributeAssignmentLocked", false)

    Derived collections (re-derived every `itemsUpdateTick`):
    - `activeSkills` = actor.items filtered by `item.type === "skill" && item.system.skillType === "active"`
    - `knowledgeSkills` = filtered by skillType === "knowledge"
    - `languageSkills` = filtered by skillType === "language"
    - Each sorted by item.name alphabetically

    Template: h1 with "Skills" heading, tab buttons (active/knowledge/language), then conditional render of SkillsActive / SkillsKnowledge / SkillsLanguage based on activeTab.

    Pass down: `actor`, `skills` (filtered array), `inCreationMode` (boolean: `!!$isCharacterCreation && !!$attributeAssignmentLocked`), `itemsUpdateTick`.

    Do NOT import Packery — skills use plain flex/grid layout, not masonry.
    Do NOT call storeManager.Subscribe here — it's already subscribed by parent (CharacterSheetApp doesn't subscribe, but DicePools does). Actually, DO subscribe: call storeManager.Subscribe(actor) in $effect and return storeManager.Unsubscribe(actor).
  </action>
  <verify>No TypeScript errors in file. File renders without crash when mounted in CharacterSheetApp.</verify>
  <done>Skills.svelte exists, exports correct props, has item reactivity, renders correct sub-component based on tab.</done>
</task>

  <name>Task 2: SkillCard.svelte — unified skill display card</name>
  <files>module/ui/actors/actor-components/skills/SkillCard.svelte</files>
  <action>
    Create unified SkillCard.svelte used by all three tab components.

    Props:
    - `actor: Actor`
    - `item: Item` (the skill item)
    - `category: "active" | "knowledge" | "language"`
    - `inCreationMode: boolean`

    Display:
    - Skill name (item.name)
    - Skill rating value: `item.system[${category}Skill].value` (cast system as Record)
    - If `category === "language"`: also show readwrite value as `R/W: ${item.system.languageSkill.readwrite.value}`
    - Specializations (read-only display): if `item.system[${category}Skill].specializations?.length > 0`, render each as a small tag/chip below the name
    - Shopping buttons (only when `inCreationMode === true`):
      - `−` button: `SkillSpendingService.Instance().canDecreaseSkill(actor, item.id)` → disabled if false
      - `+` button: `SkillSpendingService.Instance().canIncreaseSkill(actor, item.id, category)` → disabled if false
      - onclick handlers call the async service methods (no awaiting needed in handler — fire and forget)

    Layout: clipped-corners card pattern (same as AttributeCard pattern — clip-path polygon). Dark inner surface, highlight border top-left. See old LESS for reference but implement in SCSS.

    TypeScript: cast `item.system` as `Record<string, { value: number; specializations?: string[]; readwrite?: { value: number } }>` for type safety.

    Do NOT use any. Use Record casts.
  </action>
  <verify>No TypeScript errors. Card renders name + value. Shopping buttons appear only when inCreationMode=true.</verify>
  <done>SkillCard.svelte shows name, value, specializations (read-only), optional R/W row for language, conditional shopping buttons.</done>
</task>
