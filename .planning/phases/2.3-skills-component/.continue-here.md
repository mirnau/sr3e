## Open bug: sidebar not updating on name change
- `nameStore = storeManager.GetRWStore<string>(skill, "name", true)` writes to Foundry via `document.update({ name: value }, { render: false })`
- `render: false` suppresses sidebar refresh
- Tried `render: true` for root-level paths — triggered Packery re-layout but NOT sidebar
- Root cause unclear: Foundry's Items Directory may need a specific signal (collection change event?) to refresh an existing entry's name display
- This is the NEXT thing to investigate

<decisions_made>
- IStoreManager is THE ONLY API between Svelte and Foundry — no direct Hooks.on, no item.update() calls from components
- Root-level paths (isRoot: true) in GetRWStore use "name" as-is (no "system." prefix)
- SkillEditorApp uses `width: "auto"` + CSS `var(--component-width)` (17rem) to match metatype column width — NOT hardcoded pixels
- Image card is separate ItemSheetComponent from controls card
- SpecializationCard +/- buttons disabled during creation mode (Phase 3 will wire karma shopping)
- `render: false` stays in StoreManager for now — sidebar update mechanism not yet understood
</decisions_made>
