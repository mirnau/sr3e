<objective>
Debloat Attributes.svelte and Dossier.svelte by removing the $state+$effect store initialization anti-pattern and related bloat.

Purpose: These two actor components have stores assigned inside $effect to $state variables â€” the same pattern that caused the effect_update_depth_exceeded loop. DicePools.svelte is the reference implementation.
Output: Two cleaned actor components with stores as top-level const, required actor props, imported localize, no ?? after store dereferences.
</objective>

**Rules to enforce (from PROJECT.md constraints + session decisions):**
1. Stores are top-level `const` â€” never inside `$effect` or assigned to `$state`
2. `storeManager.Subscribe(actor)` at top level (not inside `$effect`)
3. `onDestroy(() => storeManager.Unsubscribe(actor))` â€” cleanup in `onDestroy`, not `$effect` return
4. `import { localize } from "../../../services/utilities"` â€” no local wrapper function
5. No `?? fallback` after a dereferenced store (`$storeName`) â€” stores always provide defaults
6. Required actor prop: `let { actor }: { actor: Actor } = $props()` â€” no `= null` default
7. `$derived` for computed boolean values â€” no `$effect` writing to `$state` for display state
8. No localize fallbacks: `localize(key)` not `localize(key ?? "fallback")`
</context>
