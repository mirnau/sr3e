Apply all rules to Health.svelte. This is the most complex component — 6 stores are currently `$state` variables initialized inside a `$effect`.

**Anti-patterns to fix:**

1. **6 stores in $state+$effect** (lines 13-18, 35-51):
   `stun`, `physical`, `penalty`, `overflow`, `miraculousSurvival`, `isAlive` are all `$state<Writable<...> | null>` assigned inside `$effect`. Move ALL of them to top-level `const`:
   ```ts
   storeManager.Subscribe(actor);
   const stun = storeManager.GetRWStore<number>(actor, "health.stun.value");
   const physical = storeManager.GetRWStore<number>(actor, "health.physical.value");
   const penalty = storeManager.GetRWStore<number>(actor, "health.penalty.value");
   const overflow = storeManager.GetRWStore<number>(actor, "health.overflow.value");
   const miraculousSurvival = storeManager.GetFlagStore<boolean>(actor, "miraculousSurvival", false);
   const isAlive = storeManager.GetRWStore<boolean>(actor, "health.isAlive");
   ```

2. **Unsubscribe in $effect cleanup** (lines 48-50): Replace with `onDestroy(() => storeManager.Unsubscribe(actor))`. The ECG destroy is already in `onDestroy` (line 99-101) — add the Unsubscribe there or keep them separate, either is fine.

3. **Required actor prop** (line 9): Change `let { actor = null } = $props()` to `let { actor }: { actor: Actor } = $props()`. Remove the outer `{#if actor && stun && physical && penalty && overflow}` guard and the `{:else}<p>Provide an actor...</p>` block.

4. **$effect guards** (lines 55-58, 84-89, 92-96): After stores are top-level, remove null guards like `if (!stun || !physical) return` — stores are always initialized. Keep the logic, remove the guards.

5. **Local localize wrapper** (lines 146-148): Remove `function localize(key: string): string { return game.i18n.localize(key); }`. Add `import { localize } from "../../../services/utilities"` at top.

6. **`?? 0` after store dereferences**: Remove `$stun ?? 0`, `$physical ?? 0`, `$overflow ?? 0`, `$penalty ?? 0` in template and effects — stores provide numeric defaults. Keep `Math.min(($overflow ?? 0) + 1, 10)` style guards in business logic only if the store could theoretically be non-numeric... actually stores always provide defaults, so these can be removed too. Keep `$overflow + 1` and `$overflow - 1`.
   - Exception: `Array.from({ length: 10 }, (_, i) => i < ($stun ?? 0))` — remove `?? 0`.
   - Keep `($stun ?? 0) < 10` in the overflow auto-fill effect → simplify to `$stun < 10`.

7. **Nullable Writable type imports**: After removing `$state<Writable<...> | null>`, remove `import type { Writable } from "svelte/store"` if no longer needed.

8. **Canvas/ECG state variables** (lines 21-24): `ecgCanvas`, `ecgPointCanvas`, `ecgContainer`, `ecgService` use `$state` for DOM binding — this is correct Svelte 5 usage for `bind:this`. Keep these as-is.

9. **`stunBoxes` and `physicalBoxes`** (lines 27-28): These are `$state<boolean[]>` updated by an `$effect` that reads `$stun` and `$physical`. Since we can now read stores directly, replace with `$derived`:
   ```ts
   const stunBoxes = $derived(Array.from({ length: 10 }, (_, i) => i < $stun));
   const physicalBoxes = $derived(Array.from({ length: 10 }, (_, i) => i < $physical));
   ```
   Remove the `$effect` that updates them (lines 54-58).

<task type="auto">
  <name>Task 2: Debloat AttributeCard.svelte</name>
  <files>module/ui/actors/actor-components/AttributeCard.svelte</files>
  <action>
Apply all rules to AttributeCard.svelte. Current anti-patterns:

1. **Stores in $derived re-runs** (lines 17-34): Stores are created inside `$derived` expressions that re-run on every reactive update, creating new store references each time:
   ```ts
   const attributeValueStore = $derived<Writable<number> | null>(
      actor ? storeManager.GetRWStore<number>(...) : null
   );
   ```
   This is wrong — stores should be created once. Since actor is always provided (required prop), move ALL store creations to top-level `const`:
   ```ts
   const attributeValueStore = storeManager.GetRWStore<number>(actor, `attributes.${attributeKey}.value`);
   const isCharacterCreation = storeManager.GetFlagStore(actor, FLAGS.ACTOR.IS_CHARACTER_CREATION, false);
   const attributeLocked = storeManager.GetFlagStore(actor, FLAGS.ACTOR.ATTRIBUTE_ASSIGNMENT_LOCKED, false);
   const isShoppingState = storeManager.GetFlagStore<boolean>(actor, FLAGS.ACTOR.IS_SHOPPING_STATE, false);
   const creationPointsStore = storeManager.GetRWStore<number>(actor, "creation.attributePoints");
   ```
   Note: AttributeCard does NOT call `storeManager.Subscribe(actor)` — it's a child component that relies on the parent (Attributes.svelte) having subscribed. This is correct — do NOT add Subscribe/Unsubscribe here.

2. **Required actor prop** (lines 10-15): Change `actor: Actor | null` → `actor: Actor`. Remove `= null` default. The parent (Attributes.svelte) always passes a non-null actor.

3. **`$derived` wrapping `isKarmaMode`** (lines 29-31): This is already using `$derived` correctly — keep it. But simplify now that stores are non-nullable:
   ```ts
   const isKarmaMode = $derived($isShoppingState && !$isCharacterCreation);
   ```

4. **`showChevrons` as `$derived(() => {...})` function** (lines 42-48): This is `$derived` wrapping a function that returns a value — the template calls `showChevrons()`. This is technically valid Svelte 5 but inconsistent. Replace with a plain `$derived` boolean:
   ```ts
   const showChevrons = $derived(
      !isDerivedAttribute && (
         ($isCharacterCreation && !$attributeLocked) || isKarmaMode
      )
   );
   ```
   Then update template from `{#if showChevrons()}` to `{#if showChevrons}`.

5. **Template fallback** (line 106): `{attributeValueStore ? $attributeValueStore : 0}` — since `attributeValueStore` is now always a `Writable<number>` (not null), simplify to `{$attributeValueStore}`.

6. **`canIncrease()` and `canDecrease()` store reads**: Lines 51-52, 55 read `$attributeValueStore` and `$creationPointsStore` to trigger reactivity. After stores are top-level, these reads still work correctly — keep them. The pattern `$attributeValueStore;` (read and discard) is intentional to trigger reactive recomputation.
  </action>
  <verify>File compiles. No `$derived` wrapping store creation. Actor type is `Actor` (not nullable). `showChevrons` is a plain boolean derived. Template uses `{$attributeValueStore}` directly.</verify>

  - `isKarmaMode` simplified without null checks
