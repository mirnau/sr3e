# Phase 3.1 Plan 01: Actor Components (Attributes + Dossier) Summary

Removed `$state`+`$effect` store initialization anti-pattern from `Attributes.svelte` and `Dossier.svelte`, aligning both with the `DicePools.svelte` reference implementation.

## Accomplishments

- Moved all store declarations to top-level `const` (out of `$effect` and `$state`)
- Moved `storeManager.Subscribe(actor)` to top level in both files
- Replaced `$effect` cleanup returns with `onDestroy(() => storeManager.Unsubscribe(actor))`
- Added `import { localize } from "../../../services/utilities"` to both files
- Removed local `localize()` wrapper function from `Attributes.svelte`
- Made `actor` prop required (`{ actor: SR3EActor }`) — removed `= null` defaults in both files
- Converted `isAwakened` from `$state(false)` + `$effect` to `$derived` in `Attributes.svelte`
- Removed `?? 0` after `$intelligence` and `$quickness` store dereferences
- Removed redundant `if (!actor)` / `if (!intelligence || !quickness)` guards
- Removed `{#if actor}` wrapper and `{:else}<p>Provide an actor...</p>{/if}` guard blocks
- Removed null guards (`if (isDetailsOpenStore)`, `if (actorNameStore)`) from `toggleDetails` and `handleActorNameChange` in `Dossier.svelte`
- Removed `import type { Readable } from "svelte/store"` (Attributes) and `import type { Writable } from "svelte/store"` (Dossier) — no longer used
- Added `import type SR3EActor from "../../../documents/SR3EActor"` to both files for proper prop typing

## Files Modified

- `module/ui/actors/actor-components/Attributes.svelte`
- `module/ui/actors/actor-components/Dossier.svelte`

## Decisions Made

- `Attributes.svelte`: The reaction calculation `$effect` was kept as legitimate business logic (it computes a derived value and writes to a store), but guards and `?? 0` fallbacks were removed since top-level stores always provide defaults.
- `Dossier.svelte`: The `{#if actor && actorNameStore && isDetailsOpenStore}` template guard was collapsed to just `<div class="dossier">` — actor is always provided and stores are always initialized at top level.

## Issues Encountered

None. The changes were straightforward mechanical transformations following the DicePools.svelte pattern.
