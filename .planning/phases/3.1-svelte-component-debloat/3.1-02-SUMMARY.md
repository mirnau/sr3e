# Phase 3.1 Plan 02: Actor Components (Health + AttributeCard) Summary

**Debloated Health.svelte (6 stores + stunBoxes/physicalBoxes) and AttributeCard.svelte (stores from $derived re-runs to top-level const).**

## Accomplishments

- **Health.svelte**: Moved all 6 `$state` stores (`stun`, `physical`, `penalty`, `overflow`, `miraculousSurvival`, `isAlive`) to top-level `const`; moved `storeManager.Subscribe(actor)` to top level; replaced `$effect` cleanup with `onDestroy(() => storeManager.Unsubscribe(actor))` alongside existing ECG destroy; removed local `localize()` wrapper and added `import { localize } from "../../../services/utilities"`; made actor prop required (`actor: SR3EActor`); removed outer `{#if actor && stun && ...}` template guard and `{:else}` fallback; converted `stunBoxes`/`physicalBoxes` from `$state` + `$effect` to `$derived`; removed all null guards on stores in effects; removed all `?? 0` after store dereferences; removed `import type { Writable }` (no longer needed).
- **AttributeCard.svelte**: Moved all 4 stores from `$derived` re-runs to top-level `const`; made actor prop required (`actor: SR3EActor`, no `= null` default); simplified `isKarmaMode` to `$derived($isShoppingState && !$isCharacterCreation)`; replaced `showChevrons = $derived(() => {...})` function with plain boolean `$derived`; updated template from `{#if showChevrons()}` to `{#if showChevrons}`; replaced `{attributeValueStore ? $attributeValueStore : 0}` with `{$attributeValueStore}`; removed `import type { Writable }`.

## Files Modified
- `module/ui/actors/actor-components/Health.svelte`
- `module/ui/actors/actor-components/AttributeCard.svelte`

## Decisions Made

- Health.svelte's `onDestroy` consolidates both `storeManager.Unsubscribe` and `ecgService?.destroy()` in a single callback — cleaner than two separate `onDestroy` calls.
- The `{#if miraculousSurvival}` wrapper around the revival icon was removed since `miraculousSurvival` is always initialized (GetFlagStore with `false` default). The icon now always renders; its visual state is controlled by `$miraculousSurvival`.
- AttributeCard does not call `storeManager.Subscribe/Unsubscribe` — it relies on the parent (Attributes.svelte) having already subscribed. This is intentional and correct.

## Issues Encountered

None. Both files followed a straightforward migration path from the documented anti-patterns to the DicePools.svelte reference pattern.
