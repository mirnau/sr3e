# Phase 3.1 Plan 03: Skills Layer + Injections Summary

**Removed `itemsUpdateTick` and manual Foundry hooks from Skills.svelte, debloated SkillCard.svelte stores from `$state`+`$effect` to top-level `const`, fixed nullable actor props across the skills layer and injection components, and cleaned `$derived(() => [...])` function-wrapping anti-pattern in AttributePointsState and SkillPointsState.**

## Accomplishments

- **Skills.svelte**: Removed `itemsUpdateTick = $state(0)`, removed entire `$effect` registering `collection.on(update/create/delete)` and `Hooks.on(updateItem/createItem/deleteItem)`. Replaced with top-level `storeManager.Subscribe(actor)` + `onDestroy(() => storeManager.Unsubscribe(actor))`. Simplified three `$derived` skill list expressions (removed `itemsUpdateTick >= 0 ? [...] : []` trick). Made actor prop required (`let { actor }: { actor: Actor }`). Removed `{:else}` fallback (no longer needed since actor is required). Removed redundant `{#if actor}` wrapper.
- **SkillCard.svelte**: Moved all 5 stores (`valueStore`, `specializationsStore`, `readWriteStore`, `isShoppingState`, `isCharacterCreation`) from `$state` variables initialized in `$effect` to top-level `const`. Added `import { onDestroy }` with both `Unsubscribe(item)` and `Unsubscribe(actor)` calls. Made actor prop required (`actor: Actor`, removed `Actor | null` and `= null`). Removed `Writable` type import (no longer needed). Simplified template: `{$valueStore}` (was `valueStore ? $valueStore : 0`), `$specializationsStore.length > 0` (was `($specializationsStore?.length ?? 0) > 0`), `{#each $specializationsStore as spec}` (was `{#each ($specializationsStore ?? []) as spec}`). Simplified condition guard: `$isShoppingState || $isCharacterCreation` (was `(isShoppingState && $isShoppingState) || (isCharacterCreation && $isCharacterCreation)`).
- **SkillsActive/Knowledge/Language.svelte**: Changed `actor: Actor | null` â†’ `actor: Actor`, removed `= null` default. These are pure wrappers with no other logic.
- **AttributePointsState.svelte**: Fixed `$derived(() => [...])` (function-returning) to `$derived([...])` (value). Updated template from `pointList()` to `pointList`. Removed `?? 0` after all store dereferences in `pointList`, `$effect`, and `showLockModal`.
- **SkillPointsState.svelte**: Same fixes â€” `$derived([...])` instead of `$derived(() => [...])`, `pointList` instead of `pointList()`, removed `?? 0` in `pointList` entries and `$effect` condition.
- **CharacterCreationManager.svelte**: Already clean â€” no changes needed.
- **ShoppingCart.svelte**: Already clean â€” no changes needed.

## Decisions Made

- **AttributePointsState `Hooks.on("closeApplication")`**: This `$effect` registers a Foundry hook to detect when a skill editor closes so the lock modal can be shown at the right time. This is legitimate UI timing logic (not duplicating StoreManager data reactivity), so it was left unchanged per rule 3.
- **`{#if actor}` wrapper in Skills.svelte**: Removed since actor is now required â€” the component can render directly without the null guard.
- **SkillCard `readWriteStore` null check preserved**: `{#if category === "language" && readWriteStore && $readWriteStore > 0}` keeps the null check on `readWriteStore` since it is genuinely `null` for non-language categories. The `?? 0` was removed but the null guard stays.
