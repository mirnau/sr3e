## Decisions Made

- `KarmaDistributionService` accesses `actor.system` via a typed local cast (same pattern as KarmaSpendingService) — avoids `as any` on the whole actor while keeping type safety for the karma fields
- `game.actors` cast via `as any` in KarmaManager since Foundry globals don't have full TS typing — consistent with existing patterns in the codebase
- No new i18n keys created — column headers use plain English strings as directed by the plan
- `$effect` in KarmaRow reads `$readyForCommit` to notify parent of reactive changes — clean dependency tracking without writing to `$state`
