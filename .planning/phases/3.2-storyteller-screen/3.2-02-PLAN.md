Create a singleton service following the existing singleton pattern (static `Instance()` getter, private constructor). Use `actor.update()` directly — no StoreManager needed in the service layer.

```ts
export class KarmaDistributionService {
  private static _instance: KarmaDistributionService;

  static Instance(): KarmaDistributionService {
    if (!KarmaDistributionService._instance) {
      KarmaDistributionService._instance = new KarmaDistributionService();
    }
    return KarmaDistributionService._instance;
  }

  private constructor() {}

  async CommitSelected(actor: Actor): Promise<void> {
    // Only commit if actor is marked ready
    if (!actor.system.karma.readyForCommit) return;

    const metatypeItem = actor.items.find((i: Item) => i.type === "metatype");
    const factor: number = metatypeItem?.system?.karma?.factor ?? 0;

    const currentLifetime: number = actor.system.karma.lifetimeKarma;
    const pending: number = actor.system.karma.pendingKarmaReward;
    const spent: number = actor.system.karma.spentKarma;

    const newLifetime = currentLifetime + pending;
    const newCeiling = Math.floor(newLifetime * factor);
    const newGoodKarma = newLifetime - spent - newCeiling;

    await (actor as any).update({
      "system.karma.lifetimeKarma": newLifetime,
      "system.karma.karmaPoolCeiling": newCeiling,
      "system.karma.goodKarma": newGoodKarma,
      "system.karma.pendingKarmaReward": 0,
      "system.karma.readyForCommit": false,
    });
  }
}
```

Note: `(actor as any).update()` is acceptable per Phase 1 decision "Type Safety over Foundry API — use @ts-expect-error or `as any` for Foundry document update calls."

If the TypeScript strict check fails on `actor.system.karma.*` access (Actor type doesn't expose system), use the same pattern as KarmaSpendingService — cast `actor.system` as needed.

const storeManager = StoreManager.Instance as IStoreManager;

onMount(() => {

**KarmaManager.svelte** — search + actor list + bulk actions.

Props: `{ actor: Actor }` — the storyteller screen actor (not used for data, just for context).

No StoreManager here — this component doesn't read storyteller screen data directly. It reads `game.actors` at render time.

State:
```ts
let delimiter = $state("");
const rowRefs = new Map<string, KarmaRowApi>();

```

Derived actor list — using $derived for filtering. `game.actors` is read once at component init (no reactivity needed for actor list changes):
```ts
const allActors = [...game.actors!.values()].filter((a: Actor) => a.type === "character");

const filteredActors = $derived(
  delimiter.length > 0
    ? allActors.filter((a: Actor) => a.name!.toLowerCase().includes(delimiter.toLowerCase()))
    : allActors
);
```

Bulk action functions:
```ts
function updateReadyState() {
  anyReady = Array.from(rowRefs.values()).some(row => row.readyForCommit);
}
async function commitSelected() {
  for (const actor of filteredActors) {
    await rowRefs.get(actor.id)?.CommitSelected();
  }
}
function selectAll() {
  filteredActors.forEach(a => rowRefs.get(a.id)?.Select());
}
function deselectAll() {
  filteredActors.forEach(a => rowRefs.get(a.id)?.Deselect());
}
```

Template:
- Search input: `<input type="text" bind:value={delimiter} placeholder="Search..." />`
- Buttons: Select All, Deselect All, Commit Selected (disabled when `!anyReady`)
- Table with thead columns: Portrait | Name | Points | Good Karma | Karma Pool | Lifetime Karma | Commit
- `{#each filteredActors as actor (actor.id)}` → `<KarmaRow {actor} onCommitStatusChange={updateReadyState} onmount={(api) => rowRefs.set(actor.id, api)} />`
- Empty state: `{:else}<p>No characters found.</p>`

Use `localize()` for column headers where keys exist in en.json. If keys don't exist, use plain English strings (do not create new i18n keys in this plan).
