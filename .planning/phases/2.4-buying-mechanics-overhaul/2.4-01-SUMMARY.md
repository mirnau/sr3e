# Phase 2.4 Plan 01: Buying Mechanics Overhaul — Routing & Pools Summary

**Routing fixed via explicit category param, Intelligence-linked cost formula restored, Int-derived knowledge/language pools calculated at attribute-lock**

## Accomplishments

- Skill editor routing fixed: `SkillEditorApp.launch()` now accepts explicit `category` param from `SkillCard`, bypassing the unreliable `skillType` field on items
- Knowledge/language editors now use Intelligence as linked attribute (hardcoded per SR3e rules) for the cost threshold formula — same `value < linkedAttr ? 1 : 2` formula as active skills
- Knowledge/language pools now calculated from final Intelligence at attribute-lock time (`Int × 5` / `floor(Int × 1.5)`) instead of priority table
- `CharacterInitializer` no longer pre-sets knowledge/language pools (they're 0 until attribute lock)

## Files Created/Modified

- `module/sheets/items/SkillEditorApp.ts` — Added `#category` field; `launch()` and `constructor` accept optional `category`; `_replaceHTML` uses `this.#category ?? skillType ?? "active"`

- `module/ui/items/skill-editor/KnowledgeSkillEditorApp.svelte` — Hardcodes Intelligence as linked attr; uses `value < linkedAttrRating ? 1 : 2` cost formula
- `module/ui/items/skill-editor/LanguageSkillEditorApp.svelte` — Same as knowledge editor
- `module/ui/actors/injections/AttributePointsState.svelte` — Adds `intelligenceStore`; writes Int-derived pools before locking attributes
- `module/services/character-creation/CharacterInitializer.ts` — Removed knowledge/language pool initialization from `updateData`

## Decisions Made

- **Hardcode Intelligence for knowledge/language editors** rather than reading from `skill.system.knowledgeSkill.linkedAttribute` — SR3e rules mandate Intelligence always; hardcoding is robust against item data issues (SkillModel defaults are unreliable)
- **Rules correction discovered at checkpoint**: Initial plan assumed flat 1 CP for knowledge/language; SR3e rules actually use the same linked-attr threshold formula — corrected before implementation
