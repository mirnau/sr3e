# Plan 2.5-01: Display Polish — Linked Attribute + Duplicate Prevention

## Objective
Two isolated improvements: (1) show each skill's linked attribute name in parentheses under the skill name in SkillCard, (2) prevent the same skill name from being drag-dropped onto an actor sheet more than once.

## Execution Context
- **Phase**: 2.5 — Character Sheet UX Polish
- **Type**: auto (proceeds to checkpoint:human-verify)
- **Risk**: Low — both changes are additive and isolated

## Context

### Files to Read First
- `module/ui/actors/actor-components/skills/SkillCard.svelte` — template target for linked attr display
- `styles/actor/components/skills.scss` — CSS target for `.skill-linked-attr` class
- `module/sheets/actors/CharacterSheet.ts` — drop override target

### Key Facts
- **Active skill linked attr**: `(item.system as Record<string, any>)?.activeSkill?.linkedAttribute` — a lowercase string like `"agility"`, `"body"`. Capitalize first letter for display.
- **Knowledge/language linked attr**: Always `"Intelligence"` — hardcoded (SR3e rule: Int is always the linked attribute for these categories)
- **Reading at mount time is safe**: Active skill's linked attribute never changes; knowledge/language is always Intelligence. No reactive store needed.
- **SkillCard has two template branches**: shopping/creation mode (`{#if ... isShoppingState ...}`) and normal mode (`{:else}`). Both branches render the same `.skill-card` structure. The linked attr must be added in BOTH branches.
- **Duplicate detection**: `_onDropItem` does not exist on `CharacterSheet` yet — inherits from `ActorSheetV2`. Override checks `(this.document as Actor).items.getName(item.name)`.
- **`ui.windows` pattern**: Already used in `SkillEditorApp.getExisting`. Same pattern for the override.
- **Item.fromDropData**: Foundry global. Use `(Item as any).fromDropData(data)` to avoid TS conflicts.
- **TypeScript**: Use `// @ts-expect-error — Foundry v13 API` on the `super._onDropItem` call if TypeScript doesn't recognize the method.

## Tasks

### Task 1: Linked Attribute Display in SkillCard

**File**: `module/ui/actors/actor-components/skills/SkillCard.svelte`

Add a `linkedAttrName` constant computed once from props (before the `$effect`):

```ts
function capitalize(s: string): string {
    return s ? s.charAt(0).toUpperCase() + s.slice(1) : "";
}

const linkedAttrName =
    category === "active"
        ? capitalize((item.system as Record<string, any>)?.activeSkill?.linkedAttribute ?? "")
        : "Intelligence";
```

In the template, add `<h4 class="skill-linked-attr no-margin">({linkedAttrName})</h4>` directly after the `<h6 class="no-margin skill-name">` element. This appears TWICE — once in the shopping-mode branch and once in the normal-mode branch. Both branches render `.skill-card` with the same internal structure.

### Task 2: Duplicate Skill Prevention

**File**: `module/sheets/actors/CharacterSheet.ts`

Add the following override method to `CharacterActorSheet` (after `_injectCreationManager`):

```ts
protected override async _onDropItem(event: DragEvent, data: Record<string, unknown>): Promise<unknown> {
    const item = await (Item as any).fromDropData(data) as Item | null;
    if (item?.type === "skill") {
        const actor = this.document as Actor;
        if (actor.items.getName(item.name!)) {
            ui.notifications?.warn(`"${item.name}" is already on this sheet.`);
            return;
        }
    }
    // @ts-expect-error — Foundry v13 _onDropItem signature not fully typed
    return super._onDropItem(event, data);
}
```

No imports needed — `Item` and `ui` are Foundry globals.
