### Key Facts — Packery

**Existing pattern** (from `CharacterSheetApp.svelte`):
- `PackeryGrid` is a component that renders `<div class="{gridPrefix}-packery-grid">` with hidden sizer divs
- It calls `setupPackery` on mount with `itemSelector`, `gridSizerSelector`, `gutterSizerSelector`
- Items need a CSS class matching `itemSelector` with a meaningful `min-width` set via CSS (setupPackery reads `getComputedStyle(firstItem).minWidth` to calculate column count)

**Three containers, two patterns**:

1. **Active skills** (`SkillsActive.svelte`):
   - Current outer div: `<div class="skills-active-container">` with CSS grid
   - Items: `.skill-attribute-group` divs (one per linked attribute, already existing class)
   - Packery approach: `<PackeryGrid gridPrefix="skills" itemSelector="skill-attribute-group">`
   - Each `skill-attribute-group` div acts as one Packery column; it contains a `.skill-group-cards` flex-wrap of SkillCards internally
   - Needed CSS: add `min-width: 13rem` to `.skill-attribute-group`

2. **Knowledge skills** (`SkillsKnowledge.svelte`):
   - Current outer div: `<div class="skill-group-cards">` with CSS grid
   - Items: each `SkillCard` renders `<div class="skill-card-container">` as its outer element
   - Packery approach: `<PackeryGrid gridPrefix="skills" itemSelector="skill-card-container">`
   - Each SkillCard's `.skill-card-container` acts as a Packery item directly — no wrapper needed
   - Needed CSS: `.skill-card-container` already has `min-width: 6.5rem` — no change needed

3. **Language skills** (`SkillsLanguage.svelte`):
   - Same as knowledge skills (identical structure, different `category` prop)
   - Same Packery approach: `<PackeryGrid gridPrefix="skills" itemSelector="skill-card-container">`

### Key Facts — Lock Modal Timing

**Current behavior** (`AttributePointsState.svelte:40-58`):
```ts
$effect(() => {
    const attrPoints = $creationPointsStore ?? 0;
    if (attrPoints === 0 && !$attributeLocked) {
        // immediately fires the lock modal
    }
});
```

**Desired behavior**: Same check, but also bail out if any `SkillEditorApp` is open for this actor. The `$effect` will re-evaluate whenever reactive dependencies change — but `ui.windows` is not reactive. The guard just needs to return early if editors are open; when the user manually closes their editor, Foundry closes it and the store that triggered this effect may re-run (on the next attribute point change). If the user already spent all points AND editors are open AND no more points change, the effect won't re-trigger. We can also store a "waiting" flag or use a timer workaround — but the simplest correct approach: **do nothing special**. The user closes their editor, and the _next time_ attrPoints hits 0 (e.g., they spend the last point again — but that can't happen) or... wait, if attrPoints are already 0, the effect won't re-run.

**Correct approach**: On the path where `attrPoints === 0 && !$attributeLocked`, check for open editors. If found, the effect needs to be able to re-check. Since `ui.windows` isn't reactive, subscribe to a fallback:
- Use `window.setTimeout` polling? No.
- Use a reactive `$derived` that tracks open editors? Not possible without Svelte store integration.

**Simplest correct approach** that doesn't require complex reactivity: Track the modal "pending" state. When attrPoints hit 0 but editors are open, set a local `$state` boolean `pendingLockModal = true`. Add a `beforeunload` or `close` hook on editors... but we don't have access to that from AttributePointsState.

**Actually workable approach**: The existing `$effect` only fires when reactive dependencies change. After attrPoints hits 0, if no editors are open, fire immediately. If editors ARE open, set a `pendingLockModal` Svelte state variable to `true`. Add a second `$effect` that watches `pendingLockModal` AND fires whenever a Foundry hook triggers — use a Foundry hook `closeApplication` that fires when any app closes:

```ts
// In $effect:
if (attrPoints === 0 && !$attributeLocked) {
    const hasOpen = hasOpenSkillEditors(actor.id!);
    if (!hasOpen) {
        showLockModal();
    } else {
        pendingLockModal = true; // remember to show when editors close
    }
}

// Register Foundry hook once (in onMount or $effect with cleanup):
const hookId = Hooks.on("closeApplication", (app: ApplicationV2) => {
    if (pendingLockModal && !hasOpenSkillEditors(actor.id!)) {
        pendingLockModal = false;
        showLockModal();
    }
});
// cleanup: Hooks.off("closeApplication", hookId)
```

Where:
```ts
function hasOpenSkillEditors(actorId: string): boolean {
    return Object.values(ui.windows as unknown as Record<string, { id?: string }>).some(
        (app) => app.id?.startsWith(`sr3e-active-skill-editor-${actorId}-`)
    );
}
```

And extract the dialog into a named async function:
```ts
async function showLockModal(): Promise<void> {
    const confirmed = await Dialog.confirm({ ... });
    if (confirmed) {
        const finalInt = $intelligenceStore ?? 1;
        $knowledgePoints = finalInt * 5;
        $languagePoints = Math.floor(finalInt * 1.5);
        await attributeLocked.update(() => true);
    }
}
```

**Svelte 5 lifecycle**: Use `$effect` for hook registration with cleanup return function. Do not use `onMount` (Svelte 5 convention prefers `$effect` for side effects with cleanup).

### Task 2: Lock Modal Timing Fix

**File**: `module/ui/actors/injections/AttributePointsState.svelte`

1. Extract the dialog logic into a named async function `showLockModal()` (to be called from two places).
2. Add a `let pendingLockModal = $state(false);` variable.
3. Add a helper function `hasOpenSkillEditors(actorId: string): boolean` (checks `ui.windows`).
4. Modify the existing `$effect` to check for open editors; set `pendingLockModal = true` if found.
5. Add a second `$effect` that registers a Foundry `closeApplication` hook and fires the modal when `pendingLockModal` is true and no editors are open. Return a cleanup function that calls `Hooks.off`.

Full updated script section:

```ts
const { actor, config = CONFIG.SR3E } = $props<{ actor: Actor; config?: any }>();
const storeManager: IStoreManager = StoreManager.Instance as IStoreManager;
const creationPointsStore = storeManager.GetRWStore<number>(actor, "creation.attributePoints");
const activePoints = storeManager.GetRWStore<number>(actor, "creation.activePoints");
const knowledgePoints = storeManager.GetRWStore<number>(actor, "creation.knowledgePoints");
const languagePoints = storeManager.GetRWStore<number>(actor, "creation.languagePoints");
const intelligenceStore = storeManager.GetRWStore<number>(actor, "attributes.intelligence.value");
const attributeLocked = storeManager.GetFlagStore(
    actor,
    FLAGS.ACTOR.ATTRIBUTE_ASSIGNMENT_LOCKED,
    false,
);

const pointList = $derived(() => [
    { value: $creationPointsStore ?? 0, text: "Attribute Points" },
    { value: $activePoints ?? 0,        text: "Active Skills"    },
    { value: $knowledgePoints ?? 0,     text: "Knowledge Skills" },
    { value: $languagePoints ?? 0,      text: "Language Skills"  },
]);

let pendingLockModal = $state(false);

function hasOpenSkillEditors(): boolean {
    return Object.values(ui.windows as unknown as Record<string, { id?: string }>).some(
        (app) => app.id?.startsWith(`sr3e-active-skill-editor-${actor.id}-`)
    );
}

async function showLockModal(): Promise<void> {
    const confirmed = await Dialog.confirm({
        title: "Complete Attribute Assignment?",
        content:
            "<p>You have spent all your attribute points.</p><p>Lock attributes and proceed to skill assignment?</p>",
    });

    if (confirmed) {
        const finalInt = $intelligenceStore ?? 1;
        $knowledgePoints = finalInt * 5;
        $languagePoints = Math.floor(finalInt * 1.5);
        await attributeLocked.update(() => true);
    }
}

// Auto-prompt when attribute points reach 0
$effect(() => {
    const attrPoints = $creationPointsStore ?? 0;
    if (attrPoints !== 0 || $attributeLocked) return;

    if (hasOpenSkillEditors()) {
        pendingLockModal = true;
        return;
    }

    pendingLockModal = false;
    (async () => { await showLockModal(); })();
});

// Re-check when any app closes — fires the pending modal if editors are now all closed
$effect(() => {
    const hookId = Hooks.on("closeApplication", () => {
        if (pendingLockModal && !hasOpenSkillEditors()) {
            pendingLockModal = false;
            (async () => { await showLockModal(); })();
        }
    });
    return () => { Hooks.off("closeApplication", hookId); };
});
```

**Important**: The second `$effect` runs once on mount and registers the hook. The cleanup unregisters it when the component unmounts. `pendingLockModal` is read inside the Foundry hook callback (not as a reactive dependency of the `$effect`), which is intentional — we don't want `$effect` to re-run every time `pendingLockModal` changes.
