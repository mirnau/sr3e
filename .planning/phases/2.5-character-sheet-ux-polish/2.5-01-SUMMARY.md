# Phase 2.5 Plan 01: Display Polish Summary

**Linked attribute shown on every SkillCard as h5 with health-blue color; duplicate skill drops blocked by name check in _onDropItem**

## Performance

- **Duration:** ~10 min
- **Tasks:** 2
- **Files modified:** 3

## Accomplishments
- Each SkillCard now displays the skill's linked attribute in parentheses below the name — active skills read from `activeSkill.linkedAttribute`, knowledge/language hardcode "Intelligence"
- Linked attr styled as `<h5>` using global VTT323-header font, scoped to `0.75rem` and `var(--health-highlight-color-primary)` (#4bd0ff) inside `.skill-card`
- `_onDropItem` override in `CharacterActorSheet` blocks drag-dropping a skill whose name already exists on the actor, with a `ui.notifications.warn` message

## Files Created/Modified
- `module/ui/actors/actor-components/skills/SkillCard.svelte` — added `capitalize()` + `linkedAttrName` constant; `<h5>` in both template branches
- `styles/actor/components/skills.scss` — added `.skill-card h5 { font-size: 0.75rem; color: var(--health-highlight-color-primary); }`
- `module/sheets/actors/CharacterSheet.ts` — added `_onDropItem` override with name-based duplicate guard

## Decisions Made
- Used `<h5 class="no-margin">` with no custom class — governed entirely by `.skill-card h5` contextual rule and global VTT323-header styles. No inline styling.
- h5 color set to `var(--health-highlight-color-primary)` (#4bd0ff) — matches the health heart icon blue, providing visual consistency across the sheet.
- Duplicate check by item name (not GUID) — per SR3e design intent: skills with the same name are the same skill regardless of item ID.

## Deviations from Plan

### Post-checkpoint Corrections
- Plan specified `<h4>` — changed to `<h5>` at user request (h4 color is highlight-color-tertiary; user preferred h5 with health-blue)
- Plan specified `.skill-linked-attr` custom class — removed in favour of bare `<h5>` governed by `.skill-card h5` rule. User policy: no inline-style-equivalent classes.
