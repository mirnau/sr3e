## Accomplishments
- All three skill tabs (active, knowledge, language) now use `PackeryGrid` with `itemSelector="skill-card-container"` — fluid Packery layout replaces old CSS grids
- Active skills grouping by linked attribute removed entirely — flat card list, same as knowledge/language
- `skill-card-container` given `width: 100%; padding: 0.125rem` — cards are 0.25rem apart in all directions via padding (gutter set to 0 so Packery and applyWidths are consistent)
- `AttributePointsState.svelte` now guards the lock modal: if any `SkillEditorApp` is open for this actor when attribute points hit 0, `pendingLockModal = true`. A `closeApplication` Foundry hook fires the modal once all editors are closed.

- `module/ui/actors/injections/AttributePointsState.svelte` — added `hasOpenSkillEditors()`, `showLockModal()`, `pendingLockModal` state, two-effect pattern with `closeApplication` hook

## Decisions Made
- **Flat layout for active skills**: Removed linked-attribute grouping — vestige of old project, incomplete in this version, adds unnecessary complexity
- **Padding over gutter for spacing**: Packery's `gutter` option reads `offsetWidth` (0 for `display: none` sizers) while `applyWidths` reads `getComputedStyle().width`. Setting gutter to `width: 0` aligns both to 0; `padding: 0.125rem` on cards gives 0.25rem visual gap reliably.
- **`closeApplication` hook pattern**: Since `ui.windows` is not reactive, a Foundry hook is the only way to re-check editor state after a close event. Hook registered in second `$effect` with cleanup.
