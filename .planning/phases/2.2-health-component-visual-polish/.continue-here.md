<decisions_made>

- **`.ecg-viewport` wrapper** — canvas elements fill their parent 100%, so they previously
  covered the 1px cyan border. Wrapping them in a 1px-inset clipped div keeps them inside
  the border area. This is the architecturally clean fix vs. tweaking clip-path math.
- **Gradients inside viewport** — gradients moved from 1px-inset (matching `::before`) to
  inside `.ecg-viewport` (also 1px-inset), eliminating the need for their own clip-paths.
  Their corner clips were simplified back to matching the container's corners.
- **No `inner-offset` on `::before`** — the mixin's `inner-offset` expands the clip polygon
  outward (toward corners), making the corner border THINNER. For a 1px border:
  - `inner-offset = 0`: corner border ≈ 1.41px (visible, slightly wider than straight edges)
  - `inner-offset = 0.1rem`: corner border ≈ 0.28px (sub-pixel, invisible — was the bug)
  - `inner-offset = 0.125rem`: corner border ≈ 0px (perfectly covered, no border at corners)
  Default (0) is correct — slight width inconsistency at corners is imperceptible.

</decisions_made>

<context>
The ECG container architecture after this session:
```
.ecg-container  (position: relative, cyan bg, clip-path octagon 0.25rem)
  ::before       (absolute, 1px inset, dark bg, clip-path octagon 0.25rem, z-index: 0)
                  → creates the cyan 1px border effect
  .ecg-viewport  (absolute, 1px inset, no bg, clip-path octagon 0.25rem, z-index: 1)
    canvas#ecg-canvas       (fills viewport 100%, z-index: 0)
    canvas#ecg-point-canvas (fills viewport 100%, z-index: 1)
    .left-gradient   (top:0 left:0, 2rem wide, full height, clip left corners, z-index: 2)
    .right-gradient  (top:0 right:0, 2rem wide, full height, clip right corners, z-index: 2)
```

The condition-monitor uses .ecg-container and .condition-monitor at margin-top: 0 !important
to suppress the sheet-card-outline's >*+* universal sibling rule (0.6rem between all children).

The `sheet-card-outline>*+*` rule in sheetcard.scss is the source of default sibling spacing —
anything that needs to override it must use `!important` or higher specificity.
</context>
