# Phase 3.4 Plan 01: SimpleStat Derived Store Consolidation Summary

**Replaced three duplicated `sumStore()` local helpers with a single `GetSimpleStatROStore(document, dataPath)` on StoreManager; fixed the silent `.modifier` → `.mod` bug that has zeroed all modifiers since the TS migration began.**

## Accomplishments

- Added `GetSimpleStatROStore` to StoreManager with a private `#simpleStatSumStores` cache — returns a derived store summing `${dataPath}.value` and `${dataPath}.mod`, keyed per `${uuid}:sum:${dataPath}`
- Removed `GetSumROStore` from StoreManager and IStoreManager; updated IStoreManager with JSDoc for the new method
- Removed local `sumStore()` helper from DicePools.svelte (8 calls → direct `GetSimpleStatROStore` calls)
- Removed local `createAttributeSumStore()` helper from Attributes.svelte (3 calls → direct method calls)
- Removed local `sumStore()` helper from Movement.svelte (3 calls → direct method calls)
- Fixed the silent `.modifier` field name bug — all three components were reading `${path}.modifier` but SimpleStat defines the field as `mod`, so modifiers were always 0

## Files Created/Modified

- `module/utilities/StoreManager.svelte.ts` - Added `GetSimpleStatROStore`, added `#simpleStatSumStores` cache + cleanup, removed `GetSumROStore`
- `module/utilities/IStoreManager.ts` - Replaced `GetSumROStore` with `GetSimpleStatROStore` with full JSDoc
- `module/ui/actors/actor-components/DicePools.svelte` - Removed `sumStore` helper, replaced 8 calls with `storeManager.GetSimpleStatROStore`
- `module/ui/actors/actor-components/Attributes.svelte` - Removed `createAttributeSumStore` helper, replaced 3 calls
- `module/ui/actors/actor-components/Movement.svelte` - Removed `sumStore` helper, replaced 3 calls

## Decisions Made

- Added `#simpleStatSumStores` cleanup to the `#cleanup` method so cached derived stores are evicted when a document unsubscribes — consistent with the lifecycle management of all other store maps
- The cache key uses `${uuid}:sum:${dataPath}` (logical path) rather than the normalized `system.` path, matching the pattern of caller-facing API

## Issues Encountered

None — build passed with zero TypeScript errors on first attempt.
