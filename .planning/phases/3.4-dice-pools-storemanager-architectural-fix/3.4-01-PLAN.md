<objective>
Replace the scattered `sumStore()` helper pattern with a single `GetSimpleStatROStore(document, dataPath)` method on StoreManager. Fixes both the architectural duplication (three components each defining a local sumStore helper) and a silent bug (all three reference `${path}.modifier` but SimpleStat defines the field as `mod`, so modifiers have never applied).

Purpose: Centralize SimpleStat value+mod summing in StoreManager where it belongs; fix the `.modifier` → `.mod` field name bug that has silently zeroed all modifiers.
Output: Updated StoreManager + IStoreManager with new method; DicePools, Attributes, Movement simplified with local helpers removed.
</objective>

**Prior decisions affecting this phase:**
- No exception handling — use non-null assertions (!), let code fail naturally
- TypeScript strict mode with no `any` types
- Stores are top-level `const` declarations — never inside `$effect` or `$state` (Phase 3.1 rule)
- Singleton pattern: `StoreManager.Instance` — all store access goes through the singleton

**The bug being fixed:**
- `SimpleStat.defineSchema()` defines `mod` (not `modifier`)
- All three components call `GetRWStore(actor, "${path}.modifier")` — path resolves to `undefined` → 0
- Result: modifiers have silently been 0 everywhere since the TS migration began

<task type="auto">
  <name>Task 1: Add GetSimpleStatROStore to StoreManager and IStoreManager</name>
  <files>
    module/utilities/StoreManager.svelte.ts,
    module/utilities/IStoreManager.ts
  </files>
  <action>
In `StoreManager.svelte.ts`:
- Add a private Map to cache sum stores: `#simpleStatSumStores = new Map<string, Readable<number>>()`
- Add method `GetSimpleStatROStore(document: FoundryDocument, dataPath: string): Readable<number>`:
  - Key: `${document.uuid}:sum:${dataPath}` (using `system.${dataPath}` form via existing normalizePath, but the key itself can just use the logical path)
  - If key exists in `#simpleStatSumStores`, return cached store
  - Otherwise: get value store via `this.GetRWStore<number>(document, `${dataPath}.value`)` and mod store via `this.GetRWStore<number>(document, `${dataPath}.mod`)` (NOT `.modifier` — the field is `mod` in SimpleStat)
  - Create a `derived([valueStore, modStore], ([$v, $m]) => ($v || 0) + ($m || 0))` store
  - Cache it in `#simpleStatSumStores` under the key and return it
- Remove `GetSumROStore` method entirely (it will have no callers after Task 2)

In `IStoreManager.ts`:
- Add `GetSimpleStatROStore(document: FoundryDocument, dataPath: string): Readable<number>` to the interface with JSDoc:
  - Documents that `dataPath` is the logical path (e.g., `"attributes.body"`) — method internally reads `.value` and `.mod` fields and returns a derived sum
- Remove `GetSumROStore(stores: Writable<number>[]): Readable<number>` from the interface
  </action>

  <done>
- `GetSimpleStatROStore` exists on both the class and interface
- Uses `.mod` (not `.modifier`) for the mod field
- Result is cached per `${uuid}:sum:${dataPath}`
- `GetSumROStore` removed from both files
  </done>
